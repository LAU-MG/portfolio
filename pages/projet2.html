Développement web front-end avec un CMS

Contexte

Pour le stage de deuxième année de BTS SIO.Je devais faire la partie front-end sur le site internet déjà créé sans aller dans le code source du site mais en passant par un CMS.

Le besoin :

Je devais remettre à neuf plusieurs pages internet voir compléter certaines qui étaient vides.

L’environnement technologique :

Le site web était codé en .NET pour la partie back-end. La partie front-end s’est faite sur le CMS en HTML et CSS, pour certaines pages, j’utilisais les outils du CMS qui généraient des éléments dans la page. La base de données était en PHP.

La réalisation :

Il y a plusieurs étapes pour la création de pages et le renouvellement de pages. Tout d’abord, il a fallu regrouper toutes les pages concernées pour ces modifications.

Le CMS possède un bloc de saisie où l’on peut mettre du HTML (et CSS dans une certaine condition). L’espace de saisie n’était pas correctement optimisé pour écrire directement, c’est pour cela que j’ai utilisé l’IDE Visual Code.

Pour les boutons, j’ai codé et puis, par la suite, pour ajouter le lien du bouton je pouvais en utilisant l’outil sur le CMS, mais j’ai préféré tout coder directement que d’utiliser le CMS.

Maintenance des pages existantes et rédaction

La partie front-end s’est faite sur le CMS en HTML et CSS, pour certaines pages j’utilisais les outils du CMS qui généraient des éléments dans la page.

EN +

J’ai effectué le SEO de chaque page en utilisant directement le CMS. Il y a un petit encadré qui permet d’entrer les méta-tags, la description et la SEO. Le CMS va de lui-même générer dans le code du site.

Fillezilla-GESTION DU SITE INTERNET

L’environnement technologique :

Le site internet était hébergé sur Plesk et FileZilla, un client FTP rapide avec une interface.

GESTION DU SITE INTERNET

L’utilisation de l’hébergeur Plesk, du dashboard du CMS, le site internet et Gmail.

J’ai fait un tour dans le site pour vérifier son état, voir différentes choses à changer :

  • Le fonctionnement des liens internes et externes du site internet
  • Les éléments du site comme les listes déroulantes
  • Le fonctionnement des formulaires d’inscription, de connexion du site
  • Les erreurs 503, 404 ….
  • Les erreurs graphiques, placement des images et du texte…
